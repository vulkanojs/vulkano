Vulkano Awesome MVC framework for Node.js

It works!

Welcome to Vulkano, your new favorite MVC framework for node.js.

Before getting started, please note: Vulkano is very oppinionated. This means, the Vulkano dev team has setup a stack of elements for you to start working rigth away without to much setup. You can change as much as you need. It is your project, anyway. Feel free to break things as needed.

Vulkano defaults

  • For loading views, Vulkano uses Nunjucks by default. Support for ejs is in the box too. To change this setting, please refer to app/config/settings.js for instructions.

  • For controllers, Vulkano uses standard Express routes. To configure the routes, go to app/config/routes.js. To make things simpler, Vulkano will populate automatic routes to every file in the app/controllers directory that follows the following convention: exampleController.js. Replace the example word with anything you need.

    Inside controllers, name your functions as follows to get automatic routing:

    • get: default method for the controller for the get http method. If you hit the example/ route, it will load the get method. This also works with any http method (you can use get, post, put, delete in the same way).
    • 'get :id': will load any route that matches example/something and will add something to req.query.id. This also works with any http method (you can use get, post, put, delete in the same way).
    • 'get hello/:id': will load any route that matches example/hello/something and will add something to req.query.id. This also works with any http method (you can use get, post, put, delete in the same way).

    To learn more about express routes, go to https://expressjs.com/en/guide/routing.html.

  • For models, Vulkano uses Mongoose. But Mongoose with a twist: if you add Example.js to the app/models folder it will attach all properties and methods to a dinamically created Mongoose model. This is for you not to be creating mongoose models again and again and make your workflow a little bit easier by only defining the things you need. Of course, this is not bulletproof and you can just create your own 'real' mongoose models anywhere you want.

Vulkano happiness bits

Vulkano also have other tools to make your workday easier:

  • Services: allows you build service libraries to use in your project. For example, a tool for automatic email proccessing, or to connect an external API, or to modify strings. It's up to you.

  • Responses: allows you build your own responses formats. Very usefull to build APIs. By default, Vulkano packages it's own response template called VSR (Vulkano standard response), that will fit most needs.

  • Custom configurations: Inside app/config you can control everything Vulkano. Just poke around. Also, if you need to create your own config files, drop them into the app/config folder and access them in your code via the global app.config.yourfile variable. Just make sure you export the data you need in your configuration file.

Front end defaults

Sure, you will need to change this. We use foundation from Zurb as the default CSS framework for Vulkano, but you can replace it (or remove it altogheter) by anything you need.

  • Serverside HTML templates are located in the app/views folder.

  • You can have partials and templates. Combine them to build your views (see Nunjucks or EJS docs to lear more). Checkout the app/views folder to see moore.

  • Also, app/views/errors you can customize error pages for 500 and 404 status codes for http requests.

  • Your public folder is located at /public. Your compiled styles and JS files will be written there.

  • If you modify sass files in client/scss while runing npm run gulp, your styles will be parsed and compiled via Gulp. Browsersync is included in the box. Checkout /gulpfile.app.js and /gulp.base.js to make this fit your needs.

  • If you modify js files in client/js while runing npm run webpack, your js files will be proccessed and packed by Webpack. Please checkout /webpack.app.js and /webpack.base.js to see more.
